AsyncDC: Asynchronous Distributed Computing

hello

Hebi Li

Definition

Brief:

An substitution of Hadoop implemented in javascript.

More specificly:

A asynchronous distributed computing system implemented in javascript, which can be used both directly in browser and deployed on server machines.

Problem

Hadoop is not good enough.

  • Not easy and cheap enough to set up
  • Cannot handle lots of concurent IO connection

Idea

  • efficiencyjavascript can run roughly as fast as C and Java on google's V8 engine.
  • Easy to set upimplement MapReduce in javascript, we can set up a node by just opening a browser and point to the server's IP address, the browser will download the script and start computing using local computing resource, CPU and memory.
    • it is super easy to set up a cluster for any use.
    • Super easy to add/delete nodes.
    • Super cheap.
    • server deploywe can deploy both server node and client node on a traditional server, by running javascript program using nodejs, an overlay of V8 for javascript interpretation.
      • unlimited IO connectionthe model is asynchronous, it can support unlimited connections without block. On the contrary, since hadoop use java, it will create a new thread and block there for a new connection, which is resource consuming and unscalable.
    • Security Consideration

Usage Senario

  • Big Company
  • Small Company
  • Individual: a immediate need
  • Platform as a Service

Comparison: Setup

Typically when a user want to set up distributed computing, he has two choice:
  • Buy from Third Party
  • Set up his own

No difference between hadoop and AsyncDC when he choose the first one.

However:

  • he need to pay for third party.
  • not in any case he want to bother to find a third party: EXAMPLE: Sen Wang hardly need any computation ability in his daily life, but one day, he need to process image, which cost 2 hours on his machine. He want to do it in 10 minutes.

If he choose to setup his own, the differences:

  • using hadoop, he need to set up OS and Hadoop and dependencies for every machine.
  • using AsyncDC, he just need to start AsyncDC in the main machine, and open browsers on as many machine as he want, No matter it's linux, windows, mac, android or ios.
